# Lead Form — Тестове завдання

## Структура проєкту

```
├── index.html   — основна сторінка лендінгу
├── style.css    — стилі (адаптивна верстка)
├── script.js    — валідація форми, маска телефону, відправка
├── send.php     — серверний обробник (проксі до Dr Tracker API)
└── README.md    — цей файл
```

## Як запустити

Проєкт потребує PHP-сервера для коректної роботи `send.php`.

**Локально (для тестування):**

```bash
cd /шлях/до/проєкту
php -S localhost:8080
```

Відкрити `http://localhost:8080` у браузері.

**На хостингу:** просто завантажити всі файли в кореневу папку домену (Apache/Nginx з PHP).

## Опис роботи форми

### Фронтенд (index.html + script.js)

1. Користувач заповнює 4 обов'язкових поля: ім'я, прізвище, email, телефон.
2. При blur (виході з поля) спрацьовує валідація з відповідним повідомленням про помилку.
3. Телефон автоматично форматується у міжнародний формат `+XX XXX XXX XXXX`.
4. Кнопка відправки блокується на час запиту (захист від повторної відправки).
5. При успіху з'являється модальне вікно "Дякуємо!".

### Бекенд (send.php)

Основна ідея — форма відправляється через `send.php`, який виступає проксі. Браузер бачить лише запит до `send.php` (Network → Fetch/XHR), реальний API-ендпоінт Dr Tracker **не видимий** клієнту.

**Послідовність дій у send.php:**

1. Приймає POST-дані з форми
2. Перевіряє honeypot-поле (захист від ботів)
3. Санітизує всі дані (`htmlspecialchars`, `filter_var`, `preg_replace`)
4. Валідує дані на стороні сервера (як бекап клієнтської валідації)
5. Формує cURL-запит до `https://tracker.my-live-tracker.com/api/` з параметрами:
   - `ApiKey`, `ApiPassword`, `CampaignID` — зашиті в PHP, не потрапляють у клієнтський код
   - `FirstName`, `LastName`, `Email`, `PhoneNumber` — з форми
   - `Language` = `es`, `IP` = IP клієнта, `Page` = реферер
6. Обробляє відповідь API:
   - `ret_code 200/201` → повертає `{"success": true}` + redirect URL якщо є
   - `ret_code 409` → "Email вже зареєстрований"
   - Інші помилки → локалізоване повідомлення іспанською
7. Повертає JSON-відповідь фронтенду

### Захист від ботів (Honeypot)

У формі є приховане поле `name="website"`, стилізоване через CSS (`position: absolute; left: -9999px`). Воно невидиме для людей, але боти часто його заповнюють. Якщо поле заповнене — і клієнт, і сервер тихо повертають фейковий успіх без реальної відправки.

## Валідація

| Поле | Правила |
|------|---------|
| Ім'я | Обов'язкове, 2-50 символів, тільки літери |
| Прізвище | Обов'язкове, 2-50 символів, тільки літери |
| Email | Обов'язкове, перевірка формату |
| Телефон | Обов'язкове, 7-15 цифр, міжнародний формат |

Валідація подвійна: клієнт (JS) + сервер (PHP).

## Адаптивність

- Desktop: форма у 2 колонки (ім'я + прізвище)
- Планшет (< 768px): навігація ховається, форма на всю ширину
- Мобільний (< 480px): зменшені відступи, адаптований шрифт

## Технології

- HTML5, CSS3 (без фреймворків)
- Vanilla JavaScript (без jQuery/бібліотек)
- PHP 7.4+ (cURL)
- Google Fonts (Inter)
